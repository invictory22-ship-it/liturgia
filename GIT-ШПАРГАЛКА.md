# Git — коротка шпаргалка (для священника)

Код на GitHub (публічний репозиторій):
**https://github.com/invictory22-ship-it/liturgia**

ЖИВИЙ САЙТ (відкривай на телефоні, працює мікрофон):
**https://invictory22-ship-it.github.io/liturgia/**
Оновлюється САМ за 1-2 хв після кожного мого коміту. На телефоні просто
онови сторінку (за потреби - жорстко). Збережи адресу в закладки.

## Навіщо це
Кожен «коміт» — збережена контрольна точка. Якщо щось зламається —
повертаємось до будь-якої з них. Це наша страховка.

## Тобі робити нічого не треба
Усі коміти роблю я під час роботи. Ця шпаргалка — щоб ти розумів,
що відбувається, і міг сам глянути історію, якщо захочеш.

## Якщо захочеш глянути сам (термінал у папці проєкту)

Історія змін:
    git log --oneline

Що змінилось зараз (ще не збережено):
    git status

## ЯК САМОМУ ВІДПРАВИТИ ПРАВКУ (через VS Code, без термінала)
Коли вніс зміну в liturgy-data.js:
1. Клікни зліва іконку Source Control (на ній цифра = скільки змін).
2. Угорі напиши коротко що змінив (напр. "правка ектенії").
3. Натисни стрілку біля галочки -> "Commit & Push" (один крок).
   (або: галочка Commit, потім Sync Changes)
4. Через 1-2 хв сайт оновиться сам. На телефоні онови сторінку.

ЗАХИСТ: перед кожним комітом автоматично перевіряється, чи файл не
зламаний (зайва кома тощо). Якщо зламаний - коміт НЕ пройде і покаже
помилку. Тоді виправ або поклич Claude. Це береже від неробочого сайту.

## ВІДКАТ (якщо щось пішло не так)
Просто скажи мені: «поверни до попередньої версії» або опиши, що зламалось.
Я знайду потрібну точку і відкочу. Вручну краще не чіпати.

## Важливо
- PDF-и НЕ в git (важкі) — лежать тільки локально в тебе.
- Резервні копії (*.bak) і тимчасові файли git ігнорує.
